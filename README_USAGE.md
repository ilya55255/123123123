# Research Information Collector - Руководство пользователя

## Описание
Система автоматизированного сбора релевантной информации из открытых интернет-источников для анализа социальных, политических, экономических процессов. Данные структурированы для использования в RAG-пайплайнах (Retrieval-Augmented Generation).

## Основные возможности

### ✓ Поддерживаемые источники
- **OpenAlex** - открытый каталог научных работ
- **arXiv** - препринты научных статей (физика, математика, CS)
- **CrossRef** - метаданные научных публикаций
- **Semantic Scholar** - научный поисковик с AI
- **Custom URLs** - пользовательские URL (с ограничениями)

### ✓ Мультиязычность
Поддержка 10+ языков:
- Английский (English)
- Русский (Русский)
- Французский (Français)
- Немецкий (Deutsch)
- Испанский (Español)
- Китайский (中文)
- Японский (日本語)
- Корейский (한국어)
- Арабский (العربية)
- Иврит (עברית)

### ✓ Структурированные данные
Каждый документ содержит:
- **ID** - уникальный идентификатор
- **Title** - заголовок
- **Authors** - список авторов
- **Date** - дата публикации
- **DOI** - цифровой идентификатор объекта
- **URL** - ссылка на источник
- **Language** - язык документа
- **Source** - источник данных
- **Abstract** - аннотация
- **full_text_chunks** - текст разбитый на чанки (500-2000 слов)
- **Files** - ссылки на PDF и другие файлы

### ✓ Экспорт данных
- JSON - полная структура данных
- CSV - табличный формат для анализа
- NDJSON - построчный JSON для стриминга

## Использование

### 1. Запуск приложения
Приложение запускается автоматически в браузере. Не требует установки сервера или базы данных.

### 2. Поиск информации

#### Основные параметры:
1. **Keywords** (обязательно)
   - Введите ключевые слова для поиска
   - Примеры: "геополитика Украина", "climate change", "quantum computing"

2. **Date From / Date To**
   - Задайте временной диапазон
   - По умолчанию: 2024-01-01 до сегодняшнего дня

3. **Languages**
   - Выберите языки для фильтрации результатов
   - Можно выбрать несколько

4. **Sources**
   - Выберите источники для поиска
   - По умолчанию выбраны все

5. **Max Results per Source**
   - Количество результатов с каждого источника (5-100)
   - По умолчанию: 20

6. **Custom URLs** (опционально)
   - Добавьте конкретные URL для анализа
   - Один URL на строку
   - Примечание: могут быть ограничения из-за CORS

### 3. Просмотр результатов

После поиска переключитесь на вкладку **Results** для просмотра:
- Список найденных документов
- Статистика по источникам и языкам
- Возможность раскрыть детали каждого документа
- Просмотр текстовых чанков

### 4. Экспорт данных

В разделе Results доступны кнопки экспорта:
- **Export JSON** - полные данные в JSON
- **Export CSV** - таблица для Excel/Google Sheets
- **Export NDJSON** - формат для стриминга

### 5. Управление данными

- **Clear All** - удалить все сохраненные документы
- Данные хранятся в localStorage браузера
- Автоматическое удаление дубликатов по URL

## Технические детали

### Архитектура
- **Frontend**: React + TypeScript + Vite
- **Styling**: Tailwind CSS
- **Storage**: Browser localStorage
- **APIs**: REST APIs научных баз данных

### Компоненты
1. **SearchForm** - форма поиска с параметрами
2. **ResultsView** - отображение результатов и статистики
3. **DocumentCard** - карточка документа с деталями
4. **SearchService** - координация поиска по источникам
5. **StorageService** - управление localStorage

### Обработка текста
- Очистка HTML тегов и специальных символов
- Чанкование с перекрытием (overlap)
- Определение языка по Unicode-диапазонам
- Извлечение ключевых слов
- Расчет релевантности

### Этичность
- Задержки между запросами (1-2 сек)
- Соблюдение Rate Limits API
- User-Agent для идентификации
- Только публичные API

## Ограничения

### Браузерная среда
- CORS ограничения для некоторых URL
- Размер localStorage (~5-10 MB)
- Нет обработки PDF в браузере (только ссылки)

### API Ограничения
- Rate limits каждого API
- Не все API предоставляют полный текст
- Некоторые источники требуют регистрации

### Рекомендации для продакшена
1. Использовать серверный proxy для обхода CORS
2. Реализовать базу данных вместо localStorage
3. Добавить очередь задач для масштабирования
4. Интегрировать PDF-парсер (pdf.js, pdfplumber)
5. Добавить кэширование результатов
6. Реализовать ротацию proxy/IP

## Устранение неполадок

### Проблема: Не найдены документы
**Решение:**
- Проверьте корректность ключевых слов
- Расширьте временной диапазон
- Выберите больше источников
- Проверьте интернет-соединение

### Проблема: Ошибка API
**Решение:**
- Подождите 1-2 минуты (rate limit)
- Уменьшите Max Results
- Проверьте доступность API

### Проблема: localStorage переполнен
**Решение:**
- Экспортируйте данные
- Нажмите Clear All
- Освободите место в браузере
